Endless Terrain Generator

An editor tool built in Unity Engine, designed to allow developers to effortlessly create custom and endlessly generating terrain systems for their games.

This project is currently a work on progress that I have recently been developing in my free time. It is a little way from finished, but I’ve decided to share the progress I have made so far and I will further update this page as I continue to work on it in the future.

Progress So Far

Using Unity’s built in perlin noise functionality, I am able generate customisable noise maps that can be layered and used to add texture to meshes. This was done by simply offsetting the Y value of each vertex position in the mesh, by the value generated with the noise map.

Similarly, I can create simple block colour textures for these meshes using a similar concept. By assinging a colour to a height value I can check the hight of each vertex and give it a colour based on which height threshold it falls into. The result of these two first steps are detailed in the screenshots below.

I have tried creating a few different environments to show off the range terrains this system can create.

To get the tiles rendering properly, the ‘player’ object is constantly checking which tiles are within a radius around its self, then hiding/showing the necessary tiles, so that only tiles that are close enough to be seen are actually being rendered at any one time.

Also, every tile stores 3 different versions of its mesh, each with varing detail. This way, the tiles that are close and within view of the player will swap out to the high detail meshes and the further tiles will use low detail meshes.

Below is a quick demonstration of this in action. You can see the wireframes of the meshes changing as they are swapped for lower detail ones.

At this current moment in time, both the terrain generation and the tile rendering dont work too well together. Attempting to calculate the all of the mesh variations, noise maps, colour maps and handle the tile rendering at the same time leaves me with a big perfomance issue.

The common solution to get around this problem is multi-threading, as I can have the rendering happen on the main thread and the terrain generation happening on a seprate thread. However this project is going to be my first experience with threading, meaning that I am learning from scratch.

I will update this page with further when I have implemented some sort of threading, and both the tile rendering and mesh creation processes can run side by side efficiently.
