Wave Function Collapse Demo

My implementation of the physics-based wave function collapse algorithm. This tool can create automated and random 2D game levels, based on a pre-determined tile set given by any user.

I got the idea for the project after finding this GitHub post by Maxim Gumin, talking the wave function collapse algorithm and how it can be used to generate random content. This sparked my interest in recreating the algorith in Unity, with the hope that one day I can use it to randomly generate levels for a 2D game of some sort.

Feel free to have a play around and try generating some maps for yourself!

Behind the scenes

In short, the application starts off by initialising an empty grid, then running through a constant loop of evaluate. propigate, collapse until each tile in the grid has been collapsed.

This evaluation function will loop through the grid finding all of the neighbouring objects surrounding each collapsed tiles, triggering them to update their list of possible connections. This does the job of propagating the necessary information out to the surrounding tiles and updating each tiles entropy value.

Here is how the entropy values are calculated for each object in the grid. When a neighouring tile gets collapsed, this code will remove all possible tiles that cannot be connected to the neighbours tile. The number of possible tiles left after this process is the entropy for this object.

Finally, after every cycle of evaluating and propagating, this function will find the object/objects in the grid with the lowest entropy value and pick one of them at random to collapse. The process then starts over. On the left is how the lowest entropy tile is selected and then on the right is the code that decides what that tile will become

There is one edge case where there are no possible connections and the object will be given an entropy value of 0. In this case there is no possible way of filling the tile, so the process has to resart from the begining. I have a few ideas of ideas ofhow I could avoid this issue, however for the purpose this was demo built it doesnt really matter if this happens. In future however, when I go to create a game around this system, I will attempt to make this improvement.
